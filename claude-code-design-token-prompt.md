# 🎨 디자인 토큰 설계 요청 프롬프트
> 새 프로젝트 시작 시 Claude Code에 첫 번째로 붙여넣는 프롬프트.
> `[ ]` 부분만 채워서 사용.

---

## 붙여넣기용 프롬프트 전문

```
이 프로젝트의 디자인 토큰을 설계해줘.
결과물은 tailwind.config.js (extend 블록) + CSS 변수 형태로 줘.

단, 각 결정마다 "왜 이렇게 했는지" 짧게 설명을 달아줘.
나중에 클라이언트나 개발자한테 내가 직접 설명할 수 있어야 해서.

---

### 프로젝트 개요
- 업종/서비스: [ 예: 스킨케어 브랜드 쇼핑몰 ]
- 타깃 유저: [ 예: 20-30대 여성, 감도 있는 소비자 ]
- 사이트 유형: [ 예: 랜딩페이지 / 쇼핑몰 / 브랜드사이트 ]

### 무드 & 방향
- 원하는 느낌 (키워드 3개): [ 예: 깨끗한, 고급스러운, 모던한 ]
- 원하지 않는 느낌: [ 예: 촌스러운, 너무 밝은, 유치한 ]
- 레퍼런스 (있으면): [ URL 또는 "없음" ]

### 컬러 힌트 (없으면 제안해줘)
- 메인 컬러 방향: [ 예: 베이지 계열 / 모르겠음 ]
- 클라이언트 기존 브랜드 컬러: [ 예: #3B82F6 / 없음 ]
- 다크/라이트 모드: [ 라이트만 / 다크만 / 둘 다 ]

### 기능 범위 (필요한 것만 체크)
컴포넌트:
- [ ] 버튼 (Solid / Ghost / Text / Icon)
- [ ] 인풋 / 폼
- [ ] 카드
- [ ] 네비게이션
- [ ] 모달
- [ ] 배지 / 태그
- [ ] 기타: [ ]

섹션:
- [ ] 히어로
- [ ] 기능/서비스 소개
- [ ] 가격표
- [ ] 후기/리뷰
- [ ] CTA
- [ ] 푸터
- [ ] 기타: [ ]

---

출력 형태:
1. 컬러 토큰 (왜 이 색을 골랐는지 이유 포함)
2. 타이포그래피 스케일 (왜 이 단계 수인지 이유 포함)
3. 간격/반경 스케일 (왜 이 베이스인지 이유 포함)
4. 버튼 variant 정의 (왜 이 종류만 쓰는지 이유 포함)
5. tailwind.config.js extend 블록 코드
6. globals.css CSS 변수 코드
```

---

## 사용 흐름

```
1. 위 프롬프트를 Claude Code 새 세션 첫 메시지로 붙여넣기
2. [ ] 채운 채로 전송
3. Claude Code가 토큰 초안 + 이유 설명 반환
4. 마음에 안 드는 부분은 "왜 이렇게 했어?" 또는 "이걸 더 따뜻하게 바꾸면?" 으로 대화
5. 확정된 tailwind.config.js를 프로젝트에 적용
6. 이후 컴포넌트 요청 시 "아까 정한 토큰 기준으로 만들어줘" 로 일관성 유지
```

