## CSS 전역 규칙

### 커서 깜빡임 방지
모든 웹 프로젝트 세팅 시 globals.css (또는 base CSS)에 반드시 포함:

```css
*:not(input):not(textarea):not([contenteditable]) {
  caret-color: transparent;
}
```

Tailwind 프로젝트면 `@layer base` 안에 넣을 것.

## UI/레이아웃 규칙

### 불필요한 UI 요소
- 장바구니 버튼 사용하지 않음 — 커머스가 아닌 프로젝트에는 넣지 말 것
- 햄버거 메뉴 + "카테고리" 버튼 조합 불필요 — nav 링크로 충분하면 제거

### 헤더 정렬
- nav 첫 번째 링크는 `pl-0`으로 컨테이너 좌측 가장자리에 정렬
- 헤더 블록 위/아래 여백은 동일하게 맞출 것 (상단 py와 하단 nav 높이 균형)

### 카테고리 분류 원칙
- 비슷한 카테고리는 합침 (쿠키+비스킷, 캔디+젤리)
- 제품이 거의 없는 단독 카테고리는 확장하여 합침 (파이→파이·케이크)
- 이모지는 카테고리 간 겹치지 않게 설정
- 이름이 긴 카테고리는 간결하게 줄일 것 (냉장디저트→디저트)
